# 本地密钥管理速查表

这份速查表用于日常维护本地加密 Secret 仓库。

## Windows 常用入口

- [管理本地密钥.bat](../管理本地密钥.bat)
- [迁移本地密钥.bat](../迁移本地密钥.bat)

底层 PowerShell 脚本：

- [manage_local_secrets.ps1](../scripts/manage_local_secrets.ps1)
- [run_secret_migration.ps1](../scripts/run_secret_migration.ps1)

## Linux 常用入口

直接使用 CLI：

```bash
python -m monitor_app.secrets_cli list
python -m monitor_app.secrets_cli set access_control.admin_password
python -m monitor_app.secrets_cli doctor
```

## 初始化主密钥

```powershell
python -m monitor_app.secrets_cli init --master-key-file .local-secrets/monitor-master-key
```

Linux 推荐：

```bash
python -m monitor_app.secrets_cli init --master-key-file /etc/monitor-secrets/monitor-master-key
```

## 一次性迁移明文 Secret

```powershell
python -m monitor_app.secrets_cli migrate --master-key-file .local-secrets/monitor-master-key
```

## 查看已有 Secret Ref

```powershell
python -m monitor_app.secrets_cli list
```

## 设置或更新 Secret

不带 `--value` 时会安全提示输入，不会直接把明文写进命令历史。

```powershell
python -m monitor_app.secrets_cli set access_control.guest_password
python -m monitor_app.secrets_cli set access_control.admin_password
python -m monitor_app.secrets_cli set access_control.session_secret
python -m monitor_app.secrets_cli set telegram.bot_token
python -m monitor_app.secrets_cli set telegram.chat_id
```

账号类 Secret Ref 规则：

- 主账号转账：
  - `main_accounts.<main_id>.transfer_api_key`
  - `main_accounts.<main_id>.transfer_api_secret`
- 子账号：
  - `accounts.<main_id>.<account_id>.api_key`
  - `accounts.<main_id>.<account_id>.api_secret`

示例：

```powershell
python -m monitor_app.secrets_cli set main_accounts.group_a.transfer_api_key
python -m monitor_app.secrets_cli set accounts.group_a.sub1.api_secret
```

## 删除 Secret

```powershell
python -m monitor_app.secrets_cli delete telegram.bot_token
```

注意：

- 删除前先确认相关配置已经不再引用该 ref
- `settings` 分页导入不支持“留空即删除”

## 一致性检查

```powershell
python -m monitor_app.secrets_cli doctor
python -m monitor_app.secrets_cli verify
```

会检查：

- 主密钥是否可解密仓库
- 配置里引用的 ref 是否缺失
- 是否存在未被任何配置引用的冗余 ref

## 当前项目常见 Secret Ref

### 访问控制

- `access_control.guest_password`
- `access_control.admin_password`
- `access_control.session_secret`

### Telegram

- `telegram.bot_token`
- `telegram.chat_id`

### Binance

- `main_accounts.<main_id>.transfer_api_key`
- `main_accounts.<main_id>.transfer_api_secret`
- `accounts.<main_id>.<account_id>.api_key`
- `accounts.<main_id>.<account_id>.api_secret`

## 使用建议

- 业务配置文件里只保留 `*_secret_ref`
- 主密钥文件不要提交到仓库
- 导入 Excel 成功后，尽快删除本地明文文件
- 旧 secret 一旦曾经明文出现过，建议后续主动轮换
